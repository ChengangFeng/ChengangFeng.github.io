2. 分析

这个动画实现起来并不复杂，掌握几个基本的自定义view的方法即可。

实现的思路分为选中状态和未选中状态

2.1 未选中的状态

未选中的状态很简单，需要绘制的有两个图形

  • 圆环
  • 勾

2.2 选中的状态

绘制选中的动画稍微复杂一点，主要包括

  1. 绘制圆环进度条
    这个简单，直接使用drawArc()即可实现

  2. 绘制向圆心收缩的动画
    这个一开始的时候想用drawArc()加上设置画笔的宽度strokeWidth来实现，不过改变的宽度是往外扩张的，所以这个想法果断放弃。
    之后，我的想法是这样的，看下图


我就打算先绘制一个黄色的背景，然后在这个图层上面绘制一个白色的圆，半径不断的缩小，直至为0，这就反过来得到了一个向中心收缩的动画，这可以叫逆转思维吧，最近看的一本书里面说到有时候反过来思考也许会有不一样的效果。

  1. 显示勾出来
    关于这个√，我在网上搜了一波，也没有明确的指明怎么画法才是标准的，所以这里可以随意发挥，自己觉得好看就行。这里直接可以使用drawLine()可以一步搞定。
  2. 最后是圆环放大再回弹的效果
    放大回弹可以使用drawArc()，配合改变画笔的宽度来实现即可

3.具体实现

3.1 确定进度圆环和钩的位置

经过上面分析，无论是选中状态还是未选中状态，进度圆环和钩的位置是不变的，所以我们先来确定圆环的位置和钩的位置

1.@Override

2.protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {

3.    super.onMeasure(widthMeasureSpec, heightMeasureSpec);

4.    ...

5.    //设置圆圈的外切矩形,radius是圆的半径，centerX，centerY是控件中心的坐标

6.    mRectF.set(centerX - radius, centerY - radius, centerX + radius, centerY + radius);

7.

8.    //设置打钩的几个点坐标（具体坐标点的位置不用怎么理会，自己定一个就好，没有统一的标准）

9.    //画一个√，需要确定3个坐标点的位置

10.    //所以这里我先用一个float数组来记录3个坐标点的位置，

11.    //最后在onDraw()的时候使用canvas.drawLines(mPoints, mPaintTick)来画出来

12.    //其中这里mPoint[0]~mPoint[3]是确定第一条线"\"的两个坐标点位置

13.    //mPoint[4]~mPoint[7]是确定第二条线"/"的两个坐标点位置

14.

15.    mPoints[0] = centerX - tickRadius + tickRadiusOffset;

16.    mPoints[1] = (float) centerY;

17.    mPoints[2] = centerX - tickRadius / 2 + tickRadiusOffset;

18.    mPoints[3] = centerY + tickRadius / 2;

19.    mPoints[4] = centerX - tickRadius / 2 + tickRadiusOffset;

20.    mPoints[5] = centerY + tickRadius / 2;

21.    mPoints[6] = centerX + tickRadius * 2 / 4 + tickRadiusOffset;

22.    mPoints[7] = centerY - tickRadius * 2 / 4;

23.}

3.2 定义变量，标记状态

既然分选中状态和未选中状态，那个绘制过程中，就必须判断当前究竟是绘制未选中的呢还是选中了的呢。

因此在这里，我定义了一个变量isChecked

3.3 绘制未选中状态

绘制过程中那些画笔就不详细说了，一开始初始化画笔最后绘制的时候调用即可

1.@Override

2.protected void onDraw(Canvas canvas) {

3.    super.onDraw(canvas);

4.    if (!isChecked) {

5.        //绘制圆环，mRectF就是之前确定的外切矩形

6.        //因为是静态的，所以设置扫过的角度为360度

7.        canvas.drawArc(mRectF, 90, 360, false, mPaintRing);

8.

9.        //根据之前定好的钩的坐标位置，进行绘制

10.        canvas.drawLines(mPoints, mPaintTick);

11.        return;

12.    }

13.}

3.4 绘制选中状态

选中状态是个动画，因此我们这里需要调用postInvalidate()不断进行重绘，直到动画执行完毕；另外，我这里用计数器的方式来控制绘制的进度。

3.4.1 绘制圆环进度条

绘制进度圆环这里，我们定义一个计数器ringCounter,峰值为360（也就是360度），每执行一次onDraw()方法，我们对ringCounter进行自加，进而模拟进度。

最后记得调用postInvalidate()进行重绘

1.//计数器

2.private int ringCounter = 0;

3.

4.@Override

5.protected void onDraw(Canvas canvas) {

6.    super.onDraw(canvas);

7.    if (!isChecked) {

8.        ...

9.        return;

10.    }

11.    //画圆弧进度，每次绘制都自加12个单位，也就是圆弧又扫过了12度

12.    //这里的12个单位先写死，后面我们可以做一个配置来实现自定义

13.    ringCounter += 12;

14.    if (ringCounter >= 360) {

15.        ringCounter = 360;

16.    }

17.    canvas.drawArc(mRectF, 90, ringCounter, false, mPaintRing);

18.    ...

19.    //强制重绘

20.    postInvalidate();

21.}

3.4.2 绘制向圆心收缩的动画

圆心收缩的动画在圆环进度达到100%的时候才进行，同理，也采用计数器circleCounter的方法来控制绘制的时间和速度

1.//计数器

2.private int circleCounter = 0;

3.

4.@Override

5.protected void onDraw(Canvas canvas) {

6.    super.onDraw(canvas);

7.    ...

8.    //在圆环进度达到100%的时候才开始绘制

9.    if (ringCounter == 360) {

10.        //先绘制背景的圆

11.        mPaintCircle.setColor(checkBaseColor);

12.        canvas.drawCircle(centerX, centerY, radius, mPaintCircle);

13.        //然后在背景圆的图层上，再绘制白色的圆(半径不断缩小)

14.        //半径不断缩小，背景就不断露出来，达到向中心收缩的效果

15.        mPaintCircle.setColor(checkTickColor);

16.        //收缩的单位先试着设置为6，后面可以进行自己自定义

17.        circleCounter += 6;

18.        canvas.drawCircle(centerX, centerY, radius - circleCounter, mPaintCircle);

19.    }

20.    //必须重绘

21.    postInvalidate();

22.}

3.4.3 绘制钩

当白色的圆半径收缩到0后,就该绘制打钩了。

绘制打钩，这里问题不大，因为在onMeasure()中已经将钩的三个坐标点已经计算出来了，直接使用drawLine()即可画出来。

1.@Override

2.protected void onDraw(Canvas canvas) {

3.    super.onDraw(canvas);

4.    ...

5.    canvas.drawCircle(centerX, centerY, radius - circleCounter, mPaintCircle);

6.    //当白色的圆半径收缩到0后，

7.    //也就是计数器circleCounter大于背景圆的半径的时候，就该将钩√显示出来了

8.    //这里加40是为了加一个延迟时间，不那么仓促的将钩显示出来

9.    if (circleCounter >= radius + 40) {

10.        //显示打钩（外加一个透明的渐变）

11.        alphaCount += 20;

12.        if (alphaCount >= 255) alphaCount = 255;

13.        mPaintTick.setAlpha(alphaCount);

14.        //最后就将之前在onMeasure中计算好的坐标传进去，绘制钩出来

15.        canvas.drawLines(mPoints, mPaintTick);

16.    }

17.    postInvalidate();

18.}

3.4.4 绘制放大再回弹的效果

放大再回弹的效果，开始的时机应该也是收缩动画结束后开始，也就是说跟打钩的动画同时进行

因为这里要放大并且回弹，所以这里的计数器我设置成一个不为0的数值，先设置成45（随意，这不是标准），然后没重绘一次，自减4个单位。

最后画笔的宽度是关键的地方，画笔的宽度根据scaleCounter的正负来决定是加还是减

1.//计数器

2.private int scaleCounter = 45;

3.

4.@Override

5.protected void onDraw(Canvas canvas) {

6.    super.onDraw(canvas);

7.    ...

8.    if (circleCounter >= radius + 40) {

9.        //显示打钩

10.       ...

11.        //显示放大并回弹的效果

12.        scaleCounter -= 4;

13.        if (scaleCounter <= -45) {

14.            scaleCounter = -45;

15.        }

16.        //放大回弹，主要看画笔的宽度

17.        float strokeWith = mPaintRing.getStrokeWidth() +

18.            (scaleCounter > 0 ? dp2px(mContext, 1) : -dp2px(mContext, 1));

19.        mPaintRing.setStrokeWidth(strokeWith);

20.        canvas.drawArc(mRectF, 90, 360, false, mPaintRing);

21.    }

22.    //动画执行完毕，就补在需要重绘了

23.    if (scaleCounter != -45) {

24.        postInvalidate();

25.    }

26.}

27.

3.6 添加点击事件

控件到这里已经基本做好了，但还不是特别的完善。

想想checkbox，它不需要暴露外部接口也能通过点击控件来实现选中还是取消选中，所以接下来要实现的就是为控件添加点击事件

先定义一个接口OnCheckedChangeListener,实现监听此控件的监听事件

1.private OnCheckedChangeListener mOnCheckedChangeListener;

2.

3.public interface OnCheckedChangeListener {

4.    void onCheckedChanged(TickView tickView, boolean isCheck);

5.}

6.

7.public void setOnCheckedChangeListener(OnCheckedChangeListener listener) {

8.    this.mOnCheckedChangeListener = listener;

9.}

接下来，初始化控件的点击事件

11. * 初始化点击事件

23.        }
